System Architecture

Block Diagram

The system architecture model designed in Model View Controller (MVC), the below figure point out these three fundamental parts of Model, View, Controller.

Model point out in red circle
The model represents the data, and does nothing else. The model does not depend on the controller or the view.

Controller point out in blue circle
The controller provides model data to the view, and interprets user operation on the data, such as insert record. The controller depends on the view and the model.

View point out in green circle
The view displays the model data, and sends user interaction between the web page and user to the controller in server side(e.g. Submit button clicks).

Details Description

About the details of the above figure, explicate each symbol as below:

The arrow is the data flow between the symbol.

Symbol

Explanation

MySQL

The data storage we use is MySQL database

DatabaseManager

A php class responsible to get connection to database, generate and execute sql statement, get back the result and communicate between database and the system.

TableNManager

A php class inheritance to DatabaseManager, as a child of it.
A table manager responsible for single one table in database.
Allowed to override the select, insert, update, delete action in parent class to fulfill the indicated table behavior and business rules.
I.e. There are parent table and child table, when perform the delete record on parent table, it should be also delete all the related records in child table.

EntityNControllerN

A pure function to extract the user input and perform CRUD operation and some process. Such as view, insert, update, delete records.
Process like calculation, generation, import, export, inquiry, reporting.
The controller must create or new the table instance to do so.
A entity may implement many controller for different users that own difference security dimension.

ConnectionManager

A stateful programming to collect the HTTP request from the client side, before to perform the request whatever it is. Check the user authentication and user authorization. Deny to perform if the user validation is fail, return the status code and error message.
If the user is valid, perform the request and return the process result.
All the data and message will format as text in JSON format return as HTTP response.

WebClient + AngularJS

Front end of the end users, The user will access the data through the web.

SecurityManager

A class responsible for security issues, login, logout, store and destroy the login session.
Check the user authentication and user authorization and return the result.

ExcelManager

A stateless program class responsible to manipulate the excel engine, also as a adapter to handle the data conversion between the excel engine and data set.

Excel Engine

A third party PHP base excel engine - PHPExcel 1.8.1, use to operate the excel read and write.

PDF Engine

A third party PHP base pdf engine - mpdf-6.1.3, PHPExcel 1.7+ use it to convert the excel instance as pdf format.
