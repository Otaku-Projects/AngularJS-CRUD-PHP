<inquiry> Directive

Inquiry merged the entry and pageview concept, inquiry can do more in rich.
Inquiry will not get data on loaded.
User may provide the record features and/or special criteria to search and filter the records.
The search result is a data set, it will display in the element, use this element for display records in a list, display summary on dashboard.


Usage
- Display a data set in a view/table without/with navigation controls like arrows, pagination
- Display a data set in select, radio, checkbox HTML element
- As pre select critical records for <process>

Arguments

Param Type Details
ng-model expression assignable expression to bind to
program-id string Name of the program id
mode string Specify the mode [fulllist | pageview | scrollview] of it, different mode has different behaviors.
display string Specify the mode [table | tile] of it, different mode has different behaviors.
pageRecordLimit number only effect on pageview and scrollview mode,
for pageview - control how many records on single page.
for scrollview - control how many records scrolled will triger the read next action.
pageinationLimit number only effect on pageview with pagination,
if set as 3, current page is 5 then < . . . 4 5 6 . . . >
if set as 5, current page is 5 then < . . 3 4 5 6 7 . . >
tool string Specify the tools [search | pageRecordLimit | display | refresh | select | pageination] on header and footer.
allowed to specify more than one, please separate the tool by space
