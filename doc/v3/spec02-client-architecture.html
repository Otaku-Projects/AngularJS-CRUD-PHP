Client Side Architecture

AngularJS Components

In this demo materials, ui-router and ocLazyLoad are used to saving the effort on manipulate,
maintenance around 20 similar pages and keeping under a better structure. With this design, the page and related AngularJs page controller loaded on route.

You may choose your own embedded style to use the following AngularJS Components.

Directive Description

About the details of the above figure, explicate each directive symbol as below:

The arrow is the data flow between the symbol.

Symbol

Explanation

<entry>

A wrapper of the HTML form element, as a instance of a entity, you may create a data record on the entry, or you may fetch the data record from pageview to entry by access the ng-model.

<pageview>

A windows to display a record set retrieved from database, the result set displayed in pagination.

<screen>

Download and render HTML template for reusable layout of <pageview>, <entry>

<editbox>

A control set to handle foreign key conceptly, auto render the <pageview> directive and auto hide in the editbox, popup the <pageview> on click to find the tuple from the foreign table.

Select a tuple to assign the FK to the field of the <entry>, you may display the further information of the selected record in the area of <editbox>.

<message>

Separate the process result message from <entry>, <import>, or <export> directive, centralize to display in here

<range>

A wapper for two <editbox> elements, provide the basic range handling. More suitable for criteria selection on report or inquiry functions.

<upload>

Upload a file to server.

<import>

Sending import action to the server for process the uploaded excel.

<export>

Sending export action to the server for export the multi-table, multi-records as excel / pdf format.

<process>

Sending process action to the server, most likely to handle more complex requirement out of above, you may customize the data in/out format for your specifiy handling.

Service Description

About the details of the above figure, explicate each service symbol as below:

The arrow is the data flow between the symbol.

Symbol

Explanation

Security

User authenticate checking, redirect the page handling.

CookiesManager

Security will involed this to access the session stored in cookies

LockManager

entry directive will involed this lock the form after submit button clicked to avoid duplicated submission and unlock after responsd returned.

TableManager

keep the table structure to reduce the duplicated HTTP request

DataAdapter

Massage the data for HTTP request and response, convert the data between client side and server side if you use client/server partially

ThemeService

According to the ui framework setting in config, generate the template URL for different directive.

TemplateService

Load the directive template via URL given by ThemeService
